Optimizations Applied

  • read_bulk() - Batch read multiple files in single HTTP request
  • stat_bulk() - Batch stat with Rust-accelerated permission checks
  • delete_bulk() - Batch delete operations
  • PostgreSQL backend - Replaced SQLite for better concurrency
  • Rate limiting disabled by default - NEXUS_RATE_LIMIT_ENABLED=false
  • L1 Memory Cache - 10,000 file content cache (up from 100)
  • L2 Disk Cache - Persistent SSD caching
  • Directory listing cache - 5s TTL for readdir
  • Content prefetch - Prefetch during readdir for sequential access
  • Skip readahead for cached - Avoid redundant network calls

Test Environment

Before: 2026-01-10 - Sequential API calls, SQLite, smaller caches

After: 2026-01-23 - Bulk API calls, PostgreSQL, optimized caching
