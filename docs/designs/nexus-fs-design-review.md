## UI Scope

Three user-facing surfaces:
1. `nexus playground` — Textual TUI file browser
2. `nexus doctor` — CLI diagnostic output
3. Documentation site — MkDocs Material

No web UI, no dashboard. Terminal-native product.

## TUI Playground Wireframe

Framework: **Textual** (in `nexus-fs[tui]` extra)

```
┌─────────────────────────────────────────────────────────────────┐
│  nexus playground                                     [q] quit  │
├──────────────────────┬──────────────────────────────────────────┤
│  MOUNTS              │  /s3/my-bucket/                          │
│  ─────────────────   │  ──────────────────────────────────────  │
│  ▶ /s3/my-bucket     │  Name              Size    Modified      │
│    /gcs/project      │  ──────────────────────────────────────  │
│    /local/data       │  📁 reports/       —       2026-03-18    │
│                      │  📁 raw-data/      —       2026-03-15    │
│  STATUS              │  📄 README.md      2.4KB   2026-03-19    │
│  ─────────────────   │  📄 config.yaml    856B    2026-03-17    │
│  s3: ● connected     │  📄 data.csv       14MB    2026-03-16    │
│  gcs: ● connected    │                                          │
│  local: ● connected  │                                          │
│                      │  ──────────────────────────────────────  │
│                      │  [↑↓] navigate  [Enter] open  [/] search │
│                      │  [c] copy path  [p] preview  [b] back    │
├──────────────────────┴──────────────────────────────────────────┤
│  3 mounts • 2 dirs, 3 files • s3://my-bucket                   │
└─────────────────────────────────────────────────────────────────┘
```

**Layout:**
- Left panel: mount list (navigation anchor) + live backend status with latency
- Right panel: file browser (directory contents with metadata)
- Bottom bar: context (mount count, file count, current URI)
- Keyboard shortcuts displayed inline

**Hierarchy:** 1st mount list (left), 2nd file list (right), 3rd status bar (bottom).

**Distinctive element:** Live backend health status with latency per mount — no other
TUI file manager shows cloud connection health.

### Keyboard Shortcuts

| Key | Action |
|-----|--------|
| ↑↓ | Navigate file list |
| Enter | Open directory / preview file |
| b | Go back (parent directory) |
| / | Search across current backend |
| c | Copy file path to clipboard |
| p | Preview file contents |
| m | Toggle mount list (when terminal < 100 cols) |
| q | Quit |

### File Preview

- Text files: syntax-highlighted via Rich (max 1MB, truncate with "Showing first 1MB of 14MB")
- Binary files: hex preview header + "Can't preview binary files"
- Empty files: "Empty file (0 bytes)"

### Search

- Triggered on Enter (not real-time — cross-backend search is too slow for keystroke)
- Results show file path + match highlight
- Partial results shown if some backends are offline

## `nexus doctor` Output Format

```
$ nexus doctor

  Nexus Environment Check
  ═══════════════════════

  Python     3.12.4                              ✓
  nexus-fs   0.1.0                               ✓
  nexus-fast 0.1.0 (Rust)                        ✓

  Backends
  ────────
  local      /usr/local/bin/nexus                ✓ available
  s3         boto3 1.34.0                        ✓ installed
    └─ AWS credentials found (profile: default)  ✓
    └─ Region: us-east-1                         ✓
  gcs        google-cloud-storage 2.14.0         ✓ installed
    └─ ADC credentials found                     ✓
  gdrive     not installed                       ○ pip install nexus-fs[gdrive]

  Mounts
  ──────
  /s3/my-bucket    s3://my-bucket    ● connected (43ms)
  /local/data      local://./data    ● connected (1ms)

  All checks passed ✓

  Tip: Mount a second backend to try cross-backend copy.
```

**Hierarchy:** 1st environment (versions), 2nd backends (installed + credentials), 3rd mounts (connectivity + latency).

**Status indicators:**
- `✓` green — check passed
- `✗` red — check failed (always followed by fix action)
- `○` dim gray — not installed (followed by install command)
- `●` green/red — connection status

**Timing:** Show latency per backend check and per mount connectivity test.

**Tip line:** One actionable suggestion based on check results (context-aware).

## User Emotional Journey

```
STEP | USER DOES                    | USER FEELS          | DESIGN SUPPORTS
-----|------------------------------|----------------------|------------------
1    | pip install nexus-fs[s3]     | "Let's try this"    | Fast install (<10s), minimal output
2    | import nexus                 | "Simple so far"     | No warnings, no config prompts
3    | fs = nexus.mount("s3://b")   | "That's it?!"       | Instant feedback: "Mounted at /s3/b ✓"
4    | fs.read("/s3/b/file.txt")    | "It actually works"  | Returns content immediately
5    | fs.ls("/s3/b/")              | "Now I get it"      | Clean Rich table output
6    | nexus playground             | "Whoa"              | TUI opens with all mounts visible
7    | Navigate + preview a file    | "This is polished"  | Syntax highlighting, smooth nav
8    | nexus doctor                 | "Professional"      | Clear health check with green dots
```

**5-second visceral:** Mount returns instantly with one-line confirmation.
**5-minute behavioral:** TUI playground feels responsive. Preview is fast. Search works.
**5-year reflective:** "Nexus is the tool that made me stop writing custom S3 glue code."

## Terminal Design Spec

### Color Palette

```
Success:  green  (Rich: [green])     — ✓ checks, ● connected
Error:    red    (Rich: [red])       — ✗ failures, ● disconnected
Warning:  yellow (Rich: [yellow])    — ⚠ deprecations
Info:     cyan   (Rich: [cyan])      — informational messages
Muted:    dim    (Rich: [dim])       — ○ not installed, secondary text
```

### Output Formatting

- Headers: Rich `Panel` with title
- Status: `✓` green, `✗` red, `○` dim (symbols always paired — color is never sole indicator)
- Tables: Rich `Table` with no borders (clean)
- Progress: Rich `Progress` with spinner
- Errors: `Error: [message]\n  → [fix action]` (always include fix action)

### Error Message Tone

- Direct, not apologetic: "Bucket not found" not "Oops! We couldn't find your bucket"
- Always include next action: "Run `nexus doctor` for details"
- No emoji in error messages (clean, professional)
- Actionable: every error tells you what to do next

## Terminal Responsive Behavior

- **Minimum:** 80x24 (standard terminal). Below → graceful message, not crash.
- **TUI < 100 cols:** Left mount panel collapses. Toggle with `m` key. Full-width file list.
- **Doctor output:** Wraps gracefully at any width (Rich handles automatically).

## Accessibility

- All output uses semantic Rich markup (not raw ANSI codes)
- Color is never the only indicator (always paired with `✓`/`✗`/`○` symbols)
- Keyboard-only navigation: all TUI actions have keyboard shortcuts (no mouse required)
- Screen reader: TUI announces current directory and file count on navigation
- Touch targets: N/A (terminal-only)

## Docs Site Information Architecture

MkDocs Material theme (existing `mkdocs.yml`).

```
Home (hero: 2-line code example, not marketing copy)
├── Getting Started
│   ├── Installation
│   ├── 5-Minute Quickstart
│   └── Mounting Backends
├── Backends
│   ├── Local Filesystem
│   ├── Amazon S3
│   ├── Google Cloud Storage
│   └── Google Drive
├── Integrations
│   ├── fsspec / pandas
│   ├── LangChain
│   ├── CrewAI
│   └── Other Frameworks
├── CLI Reference
│   ├── nexus doctor
│   └── nexus playground
```

**Anti-slop:** Hero leads with working code, not a tagline. No 3-column feature grid.
No stock illustrations. Code examples are copy-pasteable and actually run.
