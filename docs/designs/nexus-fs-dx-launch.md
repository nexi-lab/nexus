## Vision

### 10x Check

The current plan: slim `nexus-fs` PyPI package with 5-minute quickstart.

The 10x version: `nexus-fs` becomes the standard Python filesystem abstraction for AI agents. Two lines to mount any combination of S3, GCS, Google Drive, and local storage with unified permissions. Auto-credential discovery. A TUI playground for interactive browsing. fsspec compatibility (pandas, dask, HuggingFace work out of the box). Zero-config adapters for the top 5 agent frameworks. Stripe-quality documentation site.

```python
import nexus

# Two lines. That's it.
fs = nexus.mount("s3://my-bucket", "gdrive://My Drive", "local://./data")
readme = fs.read("/s3/my-bucket/README.md")
```

### Platonic Ideal

The user feels: "Why isn't everything like this?"

- Install: 2 seconds (~15 deps). Import: 1 line. Mount: 1 line. Read: 1 line.
- Credentials auto-discovered from environment (AWS profile, GCP ADC, Drive OAuth)
- `nexus playground` opens a Rich TUI — browse backends, preview files, search
- `nexus doctor` diagnoses any setup issue with actionable fix suggestions
- Framework adapters: `from nexus.integrations.langchain import NexusDocumentLoader`
- fsspec: `pd.read_csv('nexus:///s3/my-bucket/data.csv')` — zero code changes
- Docs site: interactive quickstart, API reference, recipes, architecture diagrams

## Accepted Scope (added to this plan)

Core plan (Approach A: SDK-First Slim Package):
- New `nexus-fs` PyPI package (~15 deps, standalone-only)
- `nexus.mount()` top-level API
- Permissionless standalone mode (skip ReBAC for local use)
- Basic quickstart docs

Expansions:
- Connection string sugar (`s3://bucket`, `gcs://project/bucket`, etc.)
- Auto-credential discovery (AWS chain, GCP ADC)
- `nexus playground` Rich TUI file browser
- `nexus doctor` environment diagnostic command
- Framework adapters: LangChain, CrewAI, Claude SDK, OpenAI Agents, LangGraph
- fsspec `NexusFileSystem` compatibility layer
- Full MkDocs Material documentation site with interactive quickstart + recipes

## Architecture Decisions (from CEO Review)

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Package structure | Extracted core (C) | Only way to achieve ~15 deps; forces clean kernel/services separation |
| Sync/Async API | Async-first + sync wrapper (A) | httpx model; native async for advanced users, sync for simplicity |
| Namespace strategy | nexus-ai-fs wraps nexus-fs (C) | `import nexus` always works; zero namespace conflicts |
| Jupyter compat | nest_asyncio (A) | Industry standard; Anthropic SDK, httpx use same approach |
| API surface | Minimal + escape hatch (A) | ~10 public methods; _internal for power users; graduate on demand |
| Versioning | Independent, start v0.1.0 (A) | Freedom to iterate; honest maturity signal |
