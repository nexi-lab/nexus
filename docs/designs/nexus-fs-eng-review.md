## Architecture Decisions

### Package Structure: Extracted Core (Option C)

`nexus-fs` is an extracted core package. `nexus-ai-fs` depends on it and extends it.

```
nexus-fs (slim, ~15 base deps)
  ├── src/nexus/backends/       ✓ All backends, no up-imports
  ├── src/nexus/contracts/      ✓ All contracts + moved ReBAC types
  ├── src/nexus/core/           ✓ NexusFS, MetastoreABC, etc.
  ├── src/nexus/lib/            ✓ All utilities
  ├── src/nexus/sdk/            ✓ (post-refactor: imports from contracts)
  ├── src/nexus/storage/        ✓ Storage pillars
  ├── src/nexus/config.py       ✓ (post-refactor: lazy OAuthConfig)
  └── src/nexus/__init__.py     ✓ connect() for local profiles

nexus-ai-fs (full, 70+ deps)
  ├── Everything in slim +
  ├── src/nexus/bricks/         (all optional feature bricks)
  ├── src/nexus/server/         (FastAPI + gRPC server)
  ├── src/nexus/factory/        (service orchestration)
  ├── src/nexus/raft/           (federation, Rust extensions)
  └── src/nexus/cli/            (full CLI commands)
```

### Namespace Strategy: nexus-ai-fs wraps nexus-fs (Option C)

`nexus-ai-fs` depends on `nexus-fs` and re-exports everything. Installing `nexus-ai-fs`
supersedes `nexus-fs` entirely. `import nexus` always works regardless of which is installed.

### Sync/Async: Async-First + Sync Wrapper (Option A)

Core API is `await nexus.mount()`. Sync wrapper at `nexus.sync.mount()`.

Tiered event loop strategy:

```python
# nexus/_sync.py
def run_sync(coro):
    """Run async coroutine synchronously.

    Strategy:
    1. No running loop? → asyncio.run() (simplest path)
    2. Running loop detected (Jupyter)? → nest_asyncio.apply() + run_until_complete()
    """
    try:
        loop = asyncio.get_running_loop()
    except RuntimeError:
        return asyncio.run(coro)

    # Running loop detected (Jupyter, IPython)
    nest_asyncio.apply(loop)
    return loop.run_until_complete(coro)
```

### Permissions: Passthrough in Slim, ReBAC in Full

`nexus-fs` standalone delegates to backend-native auth:
- S3: AWS IAM decides access (your credentials = your permissions)
- GCS: GCP IAM decides
- Local: OS filesystem permissions decide

Upgrade to `nexus-ai-fs` adds unified cross-backend ReBAC.

### Google Drive: Deferred to Extras

Drive requires OAuth consent flow (browser, token storage, refresh). Not "auto-discovered"
like AWS/GCS. Ships as `nexus-fs[gdrive]` with explicit `nexus auth gdrive` setup step.

S3 + GCS + Local are the "zero-config" story at launch.

### Versioning: Independent, Starting at v0.1.0

`nexus-fs` has its own semver lifecycle. `nexus-ai-fs` pins `nexus-fs>=0.1.0,<1.0`.

### Shipping: Big Bang

All components ship in one release (extraction + URIs + backends + TUI + doctor +
fsspec + adapters + docs).

## URI Grammar Specification

```
<scheme>://<authority>/<path>

Schemes:
  s3://     → Amazon S3
  gcs://    → Google Cloud Storage
  local://  → Local filesystem
  gdrive:// → Google Drive (extras only)

Examples:
  s3://my-bucket           → mount at /s3/my-bucket/
  s3://my-bucket/subdir    → mount at /s3/my-bucket/ (root=subdir)
  gcs://project/bucket     → mount at /gcs/bucket/
  local://./data           → mount at /local/data/
  local:///tmp/nexus       → mount at /local/tmp-nexus/

Mount point override:
  nexus.mount("s3://bucket", at="/my-data")

Collision rules:
  - Two mounts with same scheme+authority: error ("already mounted")
  - Reserved paths: /__sys__/ (kernel), /__pipes__/ (internal)

Escaping:
  - Standard RFC 3986
  - Spaces: URL-encode (%20) or quote
```

## Public API Surface: Minimal + Escape Hatch

Public exports (~10 methods):
- `nexus.mount()`, `nexus.sync.mount()`
- `fs.read()`, `fs.write()`, `fs.ls()`, `fs.copy()`, `fs.delete()`
- `fs.grant()` (passthrough to backend IAM)

Everything else is `nexus._internal.*` with no stability guarantees.
Graduate APIs from internal to public based on demand.

## Error & Rescue Map

```
METHOD/CODEPATH              | WHAT CAN GO WRONG              | EXCEPTION CLASS
-----------------------------|--------------------------------|------------------

                             | Missing AWS credentials        | CredentialError
                             | Bucket doesn't exist           | BackendNotFoundError
                             | Network unreachable            | ConnectionError
                             | Permission denied (AWS IAM)    | BackendPermissionError

                             | Project doesn't exist          | BackendNotFoundError

fs.read("/s3/bucket/file")   | File not found                 | FileNotFoundError
                             | Backend timeout                | TimeoutError
Missing optional dep         | boto3 not installed            | ImportError → helpful msg

EXCEPTION CLASS              | RESCUED?  | USER SEES
-----------------------------|-----------|------------------------------------------
NexusURIError                | Y         | "Invalid URI. Did you mean 's3://bucket'?"

BackendPermissionError       | Y         | "Access denied by AWS IAM"
```
